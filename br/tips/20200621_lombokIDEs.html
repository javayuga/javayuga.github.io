# Lombok e IDEs

O Projeto Lombok representa um avanço para evitar código repetitivo (boilerplate code).

Não perca mais aqueles minutos preciosos para escrever builders e outras coisitas mais.

Mas o Lombok só valerá a pena quando sua IDE estiver ajustada.

De outra forma, irá receber alarmes falsos de compilação.

# Intellij

No caso do Intellij (minha IDE preferida), basta instalar o plugin Lombok e ajustar a configuração de pré-processamento de anotações:

Settings > Build, Execution, Deployment > Compiler > Annotation Processors

O plugin adiciona um novo menu de contexto na opção de Refactor:

# Eclipse

No caso do bom e velho Eclipse, baixe o último JAR da Maven Central (org.projectlombok:lombok:1.18.12).

Rode o JAR com um java -jar da linha de comando.

O executável irá buscar e realizar um patch na sua instalação local da IDE.
