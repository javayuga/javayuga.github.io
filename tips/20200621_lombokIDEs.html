# Lombok and IDEs

Project Lombok is a fair alternative to avoid boilerplate code.

Don't waste any more of your precious minutes to write builders and other goodies:

But Lombok will only be worth it if your IDE is properly adjusted.

Otherwise you'll get false alarms for compilation problems.

# Intellij

In the case of Intellij (my favorite IDE), you'll need to install the Lombok plugin and enable the pre-processing of annotations:

Settings > Build, Execution, Deployment > Compiler > Annotation Processors

The plugin adds a new context menu under Refactor:

# Eclipse

In case of the good and old Eclipse, download the latest JAR from Maven Central (e.g. org.projectlombok:lombok:1.18.12).

Then simply run it with java -jar from the command line.

The executable will search and patch you local installation of the IDE.
